Abstract

Entry Point is an HR onboarding tool that will be designed and built to accommodate small businesses with initial set up and training of new employees. The projects scope will be to design and develop a simple web based application to manage all the HR features that will be needed to help automate the onboarding process. While the projects success will be determined on the timely completion of a working build to present. This application will be designed and coded by the team outlined in this document for theoretical customers. There will be no associated costs for the development of this project as it will use open source tools to build and launch.

The project will be developed using a Java Spring stack including Spring Boot, Spring Security, Spring JPA and Thymeleaf as well as a frontend stack of HTML, CSS, JavaScript and Bootstrap. The combination of these tools will be used to develop a multipage web application that can be launched in the cloud for easy customer access. The backend design will follow the Model, View, Controller design pattern as well as being N-layered design. The frontend user interface will be created and designed to be both reactive to several screen sizes and ratios while also including industry standard accessibility features.

The completion and launch of this product will satisfy the market needs of standardizing and organizing human resource onboarding for small businesses that currently do not have a solution for this need. The technology stack and implementation will aim to make the program easily accessible by being available online. Customers will be able to access their programs at all times from anywhere so long as they have an internet connection.

The Problem and Background

Running a small business takes a lot of time and effort, and most owners and managers have several tasks to deal with all at the same time. Onboarding new employees is an important task but can unfortunately get rushed through to get those new employees working as quickly as possible, especially in high turnover fields.

Entry Point is meant to be a tool to help assist small businesses with their onboarding needs as well as help to automate this process. This will give more time to the owners and managers to focus on other tasks while also ensuring that the new employees have a smooth onboarding experience.

Project Objective

Develop and launch a web based application that will help management of HR onboarding for small businesses. It will have features including logging in for both admins and users, storage of user data, tasks for users to complete, and the ability for admins to review user progress.

Project High-Level Solution

Introduction

Entry Point aims to be a human resource solution for onboarding new employees for small businesses that might not have the time, structure, or organization in place to effectively complete this task. The objective to solve this problem will be to build a web application to make access to the program fast and easy for both the businesses as well as their newly hired employees. For the administrative side which will be the main workspace for the businesses will include the ability to manage users by adding new accounts, deleting accounts, editing accounts, as well as viewing the progress of tasks for those users. On the employee’s end, they will be able to sign in and view a list of tasks to complete, upload documentation, complete forms, and submit feedback about their experience. The team will assume that all users from both the admin and user sides will have basic knowledge of how to interact with a web application and so our program will not have any interactive tutorials.

Solution

The solution that the team will be implementing will be to design and build a custom made tool that will be easy to interact with a modern design. The project will be coded using Java Spring Boot to handle the web server processes, while the frontend will be build using html, CSS, JavaScript, and bootstrap for components. Thymeleaf will be used to tie the front and backends together and render the view pages for the browser. Spring Security will be used to handle the authentication of users and route admins to their specific pages while sending users to the pages they will have access to. A MySQL database will be designed and set up to store important information such as user data as well as data about user activity when they complete tasks. Since the application will be designed as a web application it will be launched and hosted on Heroku’s platform to make it accessible online. Building this tool as a customized web application will ensure that all users will have easy access to the program and it will be available at all times from anywhere in the world. Meaning that the tool will be flexible and easy to implement for both the customers (businesses) and the users (employees).
